## 調査から得られた主要な知見

### 課題② セマンティック分割・CPS按分

- **LLMに文字数を数えさせてはいけない**（トークン≠文字数。業界共通認識）
- 正解パターン：**Length Guard（決定論計測）＋ LLM Rewriter（言い換え）の二層化**
- 超過量を構造化してフィードバックするとリトライ収束が速い
- grapheme clusterで計測するのが安全（Unicode対応）
- AppTekの実装でCPS準拠率が80%→92〜99.8%に向上（産業実績あり）
- 文字数カウントのMCPサーバが複数存在（mcp-character-counter等）

### 課題① タイムスタンプ・アライメント

- WhisperXはforced alignmentでワードレベルタイムスタンプを提供（PoC検証待ち）
- pyannote.audioが話者ダイアリゼーションの定番（講義は単一話者なのでシンプル）
