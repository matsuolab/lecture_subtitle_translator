# PoC報告：WhisperXタイムスタンプ精度 ＋ LLM句読点補正パイプライン

**実施日**: 2026年3月23〜24日
**担当**: 開発チーム
**対象動画**: 松尾研 LLM講座2025特別回「Agentic RAG」（122分14秒）

---

## 1. 目的

第三期字幕化パイプラインの中核となる以下2点の実現可能性を検証する。

1. **WhisperXの文字レベルタイムスタンプ**でセンテンス境界を±0.1秒精度で検出できるか
2. **LLM補正 + diffアライメント**で書き起こし精度を上げつつタイムスタンプを引き継げるか

---

## 2. 採用アプローチ

```
WhisperX JSON（文字レベルタイムスタンプ付き）
  ↓
① Gemini 全文一括補正（gemini-3-flash-preview）
   - 句読点追加（意味単位で「。」「、」を挿入）
   - 明らかなフィラー削除（えー / あのー / えーと のみ）
   - 誤字・誤変換修正（文脈から明らかな場合のみ）
  ↓
② diff-based アライメント（difflib.SequenceMatcher）
   - equal  → 元タイムスタンプをそのまま引き継ぐ
   - insert → 句読点に直前文字の end を付与（duration=0）
   - delete → フィラー分のタイムスタンプを捨てる
   - replace→ 誤字修正済み文字に元タイムスタンプを付け直す
  ↓
③ ギャップ検出（0.30秒）+ 句点（。？！）でセンテンス分割
  ↓
SRT出力
```

**出力上限対策**：全文が35,000文字を超える場合はギャップ境界で自動チャンク分割し、前後1,500文字のオーバーラップ文脈を付けて補正。結合後に全文でdiffアライメント。

---

## 3. 検証結果

| 項目 | 結果 |
|------|------|
| 対象動画 | 122分、316セグメント、39,819文字 |
| SRT生成 | 成功 |
| タイムズレ（目視） | おおむね許容範囲（±0.1秒程度） |
| 2モデル比較 | `gemini-3.1-flash-lite-preview` < `gemini-3-flash-preview`（flashが明確に優位） |
| 書き起こし精度 | 誤字・誤変換はある程度修正されるが、完全ではない |

### 確認できたこと

- WhisperXの**文字レベルタイムスタンプ**はセンテンス境界検出に十分な精度がある
- diffアライメントにより、LLM補正後もタイムスタンプを正確に引き継げる
- ギャップ検出（無音区間）＋句点分割の組み合わせで自然なセンテンス境界が得られる
- 全文をコンテキストとして渡すことで、セグメントをまたいだ誤字・文脈も修正可能

---

## 4. 残課題（開発フェーズで対応）

| 課題 | 概要 | 優先度 |
|------|------|--------|
| 短すぎるセグメント | チャンク境界付近で数文字のセグメントが発生し字幕表示されない | 中 |
| LLM省略・要約の抑制 | フィラー以外の内容がLLMに整理・削除されるケースあり | 高 |
| 書き起こし精度 | WhisperX自体の誤認識はLLM補正だけでは限界がある | 中 |
| 字幕の文字数制御 | 1セグメントが長すぎる/短すぎるケースの調整 | 中 |
